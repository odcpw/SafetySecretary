# Export Openability Checks

This protocol covers the DOCX, XLSX, and PDF golden fixtures generated by the Safety Secretary export spike. PPTX uploaded-template-as-base remains outside this gate and is handled by the dedicated PPTX/template bead.

## Automated Harness

Run from the target app checkout:

```bash
# Defaults to ./fixtures/exports/golden; set SAFETYSECRETARY_EXPORT_FIXTURE_DIR to override.
SAFETYSECRETARY_EXPORT_FIXTURE_DIR=/path/to/fixtures/exports/golden pnpm test:exports:openability
```

The harness reads `manifest.json`, checks every in-scope DOCX/XLSX/PDF fixture, converts each DOCX to PDF through LibreOffice headless, and validates each generated or existing PDF with `pdfinfo`.

In-scope fixtures:

| ID | File | Manual app |
| --- | --- | --- |
| F1 | `hira-report.docx` | MS Word |
| F2 | `hira-report.xlsx` | MS Excel |

The original bead text calls this set "eight fixtures" but the accepted export-spike manifest contains ten DOCX/XLSX/PDF fixtures. Use the manifest as source of truth.

## Manual MS Office / Reader Protocol

Record results in `evidence/exports/openability/<date>.md` in the control repo.

For each DOCX fixture in MS Word:

- Open the file directly, not through LibreOffice.
- Confirm there is no repair prompt, unsafe-file warning, or compatibility warning.
- Confirm headings render, tables remain intact, branding/logo placeholders appear, and the disclaimer footer is present where expected.
- Save nothing back to the fixture unless the task is an explicit generator repair.

For each XLSX fixture in MS Excel:

- Open the file directly in Excel.
- Confirm there is no repair prompt, unsafe-file warning, or compatibility warning.
- Confirm each expected sheet is present, rows are readable, filters/tables render, formulas recalculate if present, and styling is intact.
- Save nothing back to the fixture unless the task is an explicit generator repair.

For each PDF fixture in Adobe Reader and Chrome:

- Open the file directly in Adobe Reader.
- Confirm there is no damaged-file warning, pages render, text is selectable, and branding/logo placeholders are visible.
- Open the same file in Chrome.
- Confirm Chrome renders without plugin errors and page count matches Adobe Reader.

## Failure Handling

A manual failure blocks this gate until it is either fixed in the relevant generator or escalated to a follow-up bead with a stable note naming the failing fixture, app/version, operating system, exact warning text, and screenshot path if available.

LibreOffice success is useful evidence but does not replace MS Word, MS Excel, Adobe Reader, or Chrome manual checks.
